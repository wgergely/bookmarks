lists¶

The core tab widgets

lists.basemodel¶

Defines the base model used to display bookmark, asset and files items.

The model loads and reset data via the BaseModel.init_data() and BaseModel.reset_data() methods.

The model itself does not store data, but all retrieved data is cached to bookmarks.common.item_data. The interface for getting and setting data can be found in the bookmarks.common.data. However, the model is responsible for populating the data cache. See BaseModel.item_generator().

The model exposes different datasets to the view using the BaseModel.task() and BaseModel.data_type() switches. This is because file items are stored as sequence and individual items simultaneously and because the file model also keeps separate data for each task folder it encounters.

The currently exposed data can be retrieved by BaseModel.model_data(). To change/set the data set emit the BaseModel.taskFolderChanged and BaseModel.dataTypeChanged signals with their appropriate arguments.

Actually, Bookmarks loads data in two passes. The model is responsible for discovering items, but will not populate the items with all the data, and instead we offload that onto threads. The model’s associated threads are defined by overriding BaseModel.queues.

The base model implements data sorting via BaseModel.sort_data() but the filtering is done using FilterProxyModel.

reset_data(*args, force=False, emit_active=True)[source]¶

Resets the model’s internal data.

The underlying data is cached in the datacache module, so here we’ll make sure the data is available for the model to use. When the optional force flag is set, we’ll use init_data to load the item data from disk.

Otherwise, the method will check if our cached data is available and if not, uses init_data to fetch it.

class bookmarks.lists.basemodel.FilterProxyModel(parent=None)[source]¶

Proxy model responsible for filtering data for the view.

We can filter items based on the data contained in the QtCore.Qt.StatusTipRole, common.DescriptionRole and common.FileDetailsRole roles. Furthermore, based on flag values (MarkedAsArchived, MarkedAsActive, MarkedAsFavourite are implemented.)

Because of perfomarnce snags, sorting function are not implemented in the proxy model, rather in the source BaseModel.

Signals:

lists.delegate¶

The delegate used to visualise bookmark, asset and file items derived from base.BaseListWidget.

The delegate is responsible for painting the thumbnails, names, and clickable buttons of the list items. The base list widget has a number of custom features, such as, clickable in-line-buttons that the deleagate is aware of.

We’re painting text using QPainterPaths for a high-quality, aliased display but the approach comes with a huge performance hit and therefore any QPainterPath paint operation is backed by a cache (see common.delegate_paths).

class bookmarks.lists.files.FilesModel(parent=None)[source]¶

Model used to list files in an asset.

The model will load files from one task folder at any given time. The current task folder can be retrieved by task(). Switching tasks is done via emitting the taskFolderChanged signals.

The model will load the found files into two separate data sets, one listing files individually, the other groups them into sequences. See bookmarks.common.sequences for the rules that determine how sequence items are identified.

Switching between FileItems and SequenceItems is done by emitting the dataTypeChanged signal.

Note

The model won’t necessarily load all files it encounters. If the parent bookmark has a valid asset config set, certain file extension might be excluded. See the bookmarks.asset_config.asset_config for details.
