bookmarks.main¶

bookmarks.main¶

Bookmarks’ main widget.

MainWidget consist of bookmarks.topbar.TopBarWidget, bookmarks.statusbar.StatusBarWidget, and bookmarks.lists.basewidget.ListsWidget. The latter is the container for the three main list item widgets: bookmarks.lists.bookmarks.BookmarksWidget, bookmarks.lists.assets.AssetsWidget and bookmarks.lists.files.FilesWidget.

Important

MainWidget won’t be fully set up until MainWidget.initialize() is called. In standalone mode, the widget will automatically be initialized, but in embedded mode, the slot has to be called manually.

bookmarks.standalone¶

The module contains the elements used when initialized in StandaloneMode.

It defines BookmarksApp, Bookmark’s custom QApplication, and BookmarksAppWindow, a modified main.MainWidget.

Note, in EmbeddedMode, Bookmarks uses main.MainWidget as the main widget.

bookmarks.exec_()[source]¶

Opens the Bookmark application.

The method creates bookmarks.standalone.BookmarksApp, and initializes all required submodules and data.

Make sure to check the list of dependencies before running.

class bookmarks.standalone.BookmarksApp(args)[source]¶

A customized QApplication used by Bookmarks to run in standalone mode.

The app will start with OpenGL and high dpi support and initializes the submodules.

See bookmarks.exec_().

class bookmarks.standalone.BookmarksAppWindow(parent=None)[source]¶

The main application window is an adapted bookmarks.main.MainWidget that adds custom sizing and frameless mode.

Custom resizing and positioning is implemented using the mousePressEvent(), mouseMoveEvent() and mouseReleaseEvent() methods.

closeEvent(event)[source]¶

Bookmarks won’t close when the main window is closed. Instead, it will be hidden to the taskbar and a pop up notice will be shown to the user.

Custom mouse move event responsible for resizing the frameless widget’s geometry.

It identifies the dragable edge area, and sets the cursor overrides.
